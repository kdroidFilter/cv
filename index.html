Profile

Self-directed open-source engineer specializing in Kotlin Multiplatform and Compose Desktop. I work at the layer beneath the application — wrapping raw OS APIs (Win32, AppKit, D-Bus, MediaFoundation, WinRT) in clean Kotlin, bridging Rust, C++, and Go into the JVM through JNI/FFM, and shipping signed, self-updating installers across macOS, Windows, and Linux from a single codebase. My work fills the gaps the ecosystem left open, and it composes: every library is designed to be used with the others.

Flagship Work

Nucleus — FRAMEWORK

★ 304 · v2.0 · Maven Central
The native desktop platform for the JVM

A Gradle-plugin framework that turns Compose Multiplatform into a complete desktop stack: native window decorations, deep OS integration, code signing, notarization, and native installers — all from one DSL. Runs as a GraalVM native image (~0.5 s cold start, 60 MB RAM, 38 MB binary) or on the JVM with AOT cache. 30+ modules; push a tag → get signed installers in 16 packaging formats. “Electron was the pioneer — Kotlin + Compose + Nucleus is what comes next.”

ComposeMediaPlayer

★ 475
The only Compose player on native codecs — no VLC, no MPV

Every other JVM media player shells out to an external process. This one talks directly to MediaFoundation (Windows), AVPlayer (macOS/iOS), GStreamer (Linux), ExoPlayer (Android), and the browser player — one @Composable, zero external dependencies, instant launch.

Supported by JetBrains · used by Pushscroll (280K+ downloads, Google Play & App Store) & AeroDL

ComposeNativeTray

★ 398
A true native system tray — not AWT, not emulated

A declarative Compose API over the OS-level system tray, each platform on its own native backend: Go + D-Bus on Linux, C++ on Windows, native APIs on macOS.

Sponsored by ETH Zürich · used by AB Download Manager (12K★)

Engineering Approach

  • Infrastructure first. Runtimes and primitives get the same care as features.
  • No leaky abstractions. Every API is complete, honest, and hard to misuse.
  • Cross-language by design. Right language per layer, one clean Kotlin surface.
  • Ecosystem-aware. Libraries are built to compose, evolve, and be replaced.
